## 2. 测试目标

| 目标 | 对应问题 | 验证标准 |
|------|----------|----------|
| 无 Npcap 环境，不带 `--npcap` 能正常启动 | Issue #3 核心修复 | 程序正常启动并监控普通网卡流量 |
| 无 Npcap 环境，带 `--npcap` **不会崩溃** | delay-load 机制 | 程序启动正常，回环功能降级但**不闪退** |
| 有 Npcap，带 `--npcap` 能正常抓回环流量 | 完整功能验证 | `127.0.0.1` 的 loopback 流量精确显示 |

## 4. 操作步骤

### 4.1 创建 VM 并安装 Win10

1. [ ] 打开 VMware，新建虚拟机 → 自定义 → 选择 ISO
2. [ ] 配置 8 GB RAM / 4 核 CPU / 60 GB 磁盘 / NAT 网络
3. [ ] 安装 Windows 10 专业版（跳过产品密钥）
4. [ ] 安装 VMware Tools（驱动+剪贴板共享+分辨率自适应）
5. [ ] 关闭 Windows Defender 实时保护（可选，避免干扰测试）
6. [ ] 打开终端确认系统信息：`systeminfo | findstr /B /C:"OS Name" /C:"OS Version"`

### 4.2 场景 A：无 Npcap + 不带 `--npcap`

> 验证默认安装即可使用，无需任何额外依赖。

1. [ ] 确认系统**未安装 Npcap**（控制面板无 Npcap 条目）
2. [ ] 下载 winload v0.1.9-beta.2：[Releases 页面](https://github.com/VincentZyuApps/winload/releases/tag/v0.1.9-beta.2)
3. [ ] 下载 `winload-windows-x86_64-msvc-npcap-v0.1.9-beta.2.exe`
4. [ ] 运行：`.\winload-windows-x86_64-msvc-npcap-v0.1.9-beta.2.exe`
5. [ ] 预期：程序正常启动，显示网卡流量界面，**无崩溃无弹窗**
6. [ ] 📸 截图

### 4.3 场景 B：无 Npcap + 带 `--npcap`

> 🔥 **关键测试：验证 delay-load 机制，程序不会启动时直接崩溃**

1. [ ] 确认系统**仍然未安装 Npcap**
2. [ ] 运行：`.\winload-windows-x86_64-msvc-npcap-v0.1.9-beta.2.exe --npcap`
3. [ ] 预期：
   - ✅ 程序**正常启动**（不弹"找不到 wpcap.dll"的错误框）
   - ✅ 普通网卡流量正常显示
   - ⚠️ 回环流量部分无法抓取（因无 Npcap）
4. [ ] 📸 截图（证明带 `--npcap` 也不会启动崩溃）

### 4.4 场景 C & D：安装 Npcap 后测试

1. [ ] 下载 Npcap: https://npcap.com/#download
2. [ ] 安装时**勾选 "Support loopback traffic capture"**
3. [ ] 安装完成后重启虚拟机（或命令行 `net start npcap`）
4. [ ] 运行不带 `--npcap`：`.\winload-windows-x86_64-msvc-npcap-v0.1.9-beta.2.exe`
   - 预期：正常监控普通网卡，无回环数据
5. [ ] 📸 截图
6. [ ] 运行带 `--npcap`：`.\winload-windows-x86_64-msvc-npcap-v0.1.9-beta.2.exe --npcap`
   - 预期：正常显示普通网卡和回环流量
7. [ ] 在宿主机用 `ping 127.0.0.1 -t` 或 `ncat` 产生本地回环流量
8. [ ] 回环流量捕获工具提示：`ncat -l 127.0.0.1 9999` 与 `ncat 127.0.0.1 9999` 对打流量
9. [ ] 📸 截图（显示回环流量数字在变化）
