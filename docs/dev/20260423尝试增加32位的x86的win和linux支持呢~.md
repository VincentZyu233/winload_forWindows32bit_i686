# 尝试增加 32 位 x86 的 Windows 和 Linux 支持

## 背景

有几台 32 位设备需要跑 winload：
- **32 位 x86 Linux**：老师发的虚拟机文件
- **32 位 x86 Windows**：学校物理实验室的旧电脑

i686 目标可以直接加到现有的 CI matrix 里，用 stable 工具链，无需 Docker 或特殊配置。

## 不做的事情

- **Scoop**：不提供 i686，特定机器直接从 Release 下载
- **AUR**：不提供 i686
- **DEB / RPM**：不提供 i686
- **npm**：不提供 i686（否则要新建 `@vincentzyuapps/winload-win32-ia32` 等包，改 `index.js` 映射表，工作量大且没必要）
- **MIPS**：继续在 fork 仓库维护

## 改动清单（共 2 个文件）

### 1. `rust/src/loopback.rs` — AtomicU64 → AtomicU32

32 位目标不支持 `AtomicU64`。这个计数器只在 Windows Npcap 回环捕获里使用，非 Windows 直接返回 `Err`，改 `AtomicU32` 对所有现有平台无影响。

具体改动（共 8 处）：

| 行号 | 改前 | 改后 |
|------|------|------|
| 7 | `AtomicU64` | `AtomicU32` |
| 17-18 | `Arc<AtomicU64>` | `Arc<AtomicU32>` |
| 24-25 | `AtomicU64::new(0)` | `AtomicU32::new(0)` |
| 30, 34 | `.load(Ordering::Relaxed)` | `.load(Ordering::Relaxed) as u64` |
| 177 | `as u64` | `as u32` |

公开接口返回类型不变（仍然是 `u64`），其他文件零改动。

### 2. `.github/workflows/build.yml` — 3 处修改

#### a) matrix 新增 2 个目标

在 windows-x86_64 后面加：
```yaml
          # ── Windows i686 (32-bit) ──
          - target: i686-pc-windows-msvc
            os: windows-latest
            name: windows-i686
            binary: winload.exe
            asset: winload-windows-i686.exe
```

在 linux-x86_64 后面加：
```yaml
          # ── Linux i686 (32-bit) ──
          - target: i686-unknown-linux-gnu
            os: ubuntu-latest
            name: linux-i686
            binary: winload
            asset: winload-linux-i686
            cross_packages: gcc-multilib
```

#### b) Npcap SDK 架构选择（~第 278 行）

加一个 i686 → x86 的分支：
```powershell
if ($target -like "*aarch64*") { $arch = "ARM64" }
elseif ($target -like "*i686*") { $arch = "x86" }
else { $arch = "x64" }
```

#### c) Release Notes 下载表格（~第 430 行后）

新增一行，只有 Windows 和 Linux，macOS 和 Android 写 `—`：
```
| **x86 (32-bit)** | [exe](...i686...) | [binary](...i686...) | — | — |
```

注意：不包含 deb/rpm 链接，因为不提供 i686 的 DEB/RPM 包。

### 额外：跳过 i686 的 DEB/RPM 打包

现有 workflow 里 DEB/RPM 的条件是：
```yaml
if: contains(matrix.target, 'linux') && !contains(matrix.target, 'android')
```

这会自动匹配 i686-linux。需要加一个排除条件：
```yaml
if: contains(matrix.target, 'linux') && !contains(matrix.target, 'android') && !contains(matrix.target, 'i686')
```

涉及 3 处（Build DEB & RPM、Upload DEB artifact、Upload RPM artifact）。

## 验证方式

1. commit message 包含 `build action`，触发构建但不发 Release
2. 检查 10 个 matrix job 全部通过（原来 8 个 + 新增 2 个）
3. 确认 i686 Linux 没有触发 DEB/RPM 打包步骤
4. 下载 artifact 确认是 32 位二进制
